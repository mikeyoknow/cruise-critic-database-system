# Project Summary

## Cruise Critic Database System

This project designs and implements a normalized relational database for a cruise-planning platform inspired by Cruise Critic.

The system stores cruise-related operational data such as ships, cruise lines, itineraries, ports, departure dates, partner deals, price tiers, bonus offers, customers, reviews, and category-based ratings.

## Functional Scope

The database supports:

- Searching cruises by itinerary, port, departure date, cruise line, and duration
- Comparing cruises using pricing, ratings, and ship information
- Tracking partner deals and cabin price tiers
- Recording customer reviews and category ratings
- Reporting on cruise popularity, rating quality, and operational characteristics

## Technical Contribution

The implementation demonstrates:

- Relational schema design
- Normalization of repeated business concepts
- SQL Server table creation
- Primary and foreign key constraints
- Junction-table modelling
- Sample data loading
- Management-style SQL reporting queries
